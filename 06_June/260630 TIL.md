### 새로 배운 내용: LangGraph 고급

### Persistence(영속성)
- 영속성은 상태를 유지(Stateful)하는 기능이다. 
#### Checkpointer
- 체크포인터는 영속성을 실행하는 저장 인터페이스다.
- *상태 유지 기능이 동작하도록 상태를 저장하고 다시 불러오는 구성 요소
- 구현체 종류
	- InMemorySaver
	- SqliteSaver
	- PostgresSaver
#### Checkpoint
- 체크포인트는 상태 기록이다.
- 필요한 이유: 디버깅
#### Thread
- (랭그래프의) Thread는 실행을 구분하는 단위다.
- 동시 실행이 아니라 State가 이어지는 실행 흐름을 구분하는 개념
#### Time Travel
- 체크포인트를 기준으로 실행을 이어가거나 다른 입력으로 재실행하는 LangGraph 기능이다.

### Durable Execution
- 실행이 끊겨도 체크포인트를 통해 같은 흐름을 이어서 실행할 수 있게 하는 실행 모델이다.
- 핵심 요소
    - Checkpointer
    - Checkpoint
    - Thread
- Durable Execution이 안정적으로 동작하기 위한 중요한 설계 원칙
    - Determinism
    - Idempotency
    - Side Effects
#### Determinism
- 같은 조건에서 다시 실행했을때 결과가 달라지지 않아야 한다.
- 그래프가 같은 입력,State,코드로 재실행했을 때 같은 결과를 재현할 수 있게 하는 일관성 원칙이다.
#### Idempotency
- 같은 작업이 여러 번 실행되어도 한번 실행한 것과 동일한 결과를 보장하는 성질이다.
- e.g) 엘리베이터 문 닫기: 한번 누르든, 여러번 누르든 결과는 문이 닫힌다는 것 하나
#### Side Effect 처리
함수를 호출했을 때 발생하는
- 주 효과: 반환 값
- 부수 효과: 반환 값이 아닌 것(발생하는 모든 것)
- 외부에 영향을 주는 작업을 안전하게 다루기 위한 설계 원칙
- 핵심
    - Side Effect를 워크플로우 본체에 그대로 두지 않고, 별도 @task 나 노드로 격리하는 것
