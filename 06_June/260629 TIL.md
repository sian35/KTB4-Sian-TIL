### 새로 배운 내용: LangGraph

### AI Agent 개념
#### LangGraph
- 랭그래프란 그래프 구조로 워크플로우를 설계하는 에이전트 프레임웍이다.
#### Workflow
- 워크플로우는 상태를 단계적으로 처리하는 구조를 의미한다.
#### AI Agent
- AI Agent란 LLM이 다음 행동을 선택하고, 도구를 호출하는 실행 구조를 의미한다.
#### GraphAPI
- 상태 스키마,노드,엣지를 이용해 그래프 구조를 구축하는 랭그래프의 기본 API다.
---
### LangGraph 기초 구조
#### StateGraph
- StateGraph란 상태 스키마를 바탕으로 빌더 객체를 만들어내는 클래스다.
#### Graph State Schema
- 그래프 상태 스키마는 랭그래프 노드에서 공유되는 키, 타입, 규칙을 선언한 상태 구조 정의(클래스)이다.
#### Reducer
- 상태 키(State Key)에 들어오는 업데이트를 기존 값과 합치는 방법을 정의하는 함수다.
#### MessagesState
- 메시지 기록을 자동으로 누적하고 관리하는 기본 상태 스키마다.
#### Node
- 현재 상태를 입력으로, 작업을 수행하고 업데이트할 값을 반환하는 함수다.
#### Edge
- 다음으로 실행할 노드나 종료 지점을 지정하는 그래프 연결이다.
#### Compile
- 빌더 객체(StateGraph)를 실행 가능한 그래프 객체로 변환하는 메서드다.

### LangGraph 핵심
#### Tool Calling
- Tool Calling이란 입력을 바탕으로 호출해야 할 도구의 이름과 인자를 반환하는 기능이다.
#### Tool Node
- Tool Calling에서 판단한 도구를 실제로 수행하고, 결과를 메시지로 변환해 메시지 리스트에 추가하는 사전 구성 노드이다.
#### Routing
- 입력이나 상태에 따라 다음으로 실행할 노드를 선택하는 분기 설계다.
#### Parallelization
- 하나의 노드로부터 여러 노드로 병렬 실행하고 그 결과를 취합하는 패턴이다.
#### Send
- 동일한 노드를 여러 번 병렬 실행하도록 하는 동적 분기 객체다.
#### Command
- 상태 갱신과 다음 노드 지정을 한번에 할 수 있는 랭그래프 객체다.
#### Loop
- 종료 조건이 만족될 때까지 반복하는 랭그래프 구조다.
#### Loop Termination
- 종료 방법을 설계하는 구조다.
#### ReAct Pattern
- 추론하고 실행한 결과를 바탕으로 반복하는 에이전트의 실행 구조다.
#### Functional API
- 함수 호출 흐름을 work flow로 실행하는 API다.
